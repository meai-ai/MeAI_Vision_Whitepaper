# MeAI 配套文件大纲 / Companion Documents Outline

> 本文件是 MeAI Vision Whitepaper v1.1 的配套规划。每个文件对应白皮书中标记为"需要验证"或"还不知道"的关键领域。

## 1. 经济可持续性备忘录 / Economic Sustainability Memo

**对应白皮书章节：** 15.9 #1（经济可持续性）、12.3（Agent持续运行成本）

**目标：** 提出2-3个可行的经济模型，带数据估算，供社区讨论。

### 大纲

1. **成本模型**
   - 单个MeAI agent的日均/月均运营成本拆解（推理、存储、网络通信）
   - 成本随模型迭代的下降曲线预测（基于过去24个月的趋势）
   - 混合架构（小模型日常 + 大模型深度思考）的成本优化估算

2. **收入模型选项**
   - 方案A：分层订阅（基础免费 + 高级付费），附公平性分析
   - 方案B：社区节点运营者激励（运营节点获得网络credit）
   - 方案C：价值交换抽成（AI间知识交易的微额费用）
   - 每个方案的优劣对比表

3. **弱势群体接入保障**
   - 补贴机制设计（谁出钱、怎么分配）
   - 超轻量agent的最低硬件要求
   - 社区代托管的经济激励

4. **盈亏平衡分析**
   - 不同用户规模下的可持续性测算
   - 与现有AI companion平台的单位经济对比

## 2. MAIP技术架构文档 / MAIP Technical Architecture Document

**对应白皮书章节：** 7.1（四步走策略）、6.2（去中心化三层面）、10.11-10.13（协议级机制）

**目标：** 将白皮书中的协议描述转化为可实现的技术规范草案。

### 大纲

1. **协议总览**
   - MAIP协议栈分层图（身份层、通信层、社交层、经济层）
   - 与现有协议的关系（ActivityPub、DID、MCP）
   - 设计原则：model-agnostic、最小权限、渐进式去中心化

2. **身份与密钥管理**
   - DID方案选型（did:key / did:web / did:peer 对比）
   - 密钥生命周期管理（生成、轮换、恢复、撤销）
   - 唯一主体原则的技术实现（活跃实例验证机制）

3. **Agent间通信协议**
   - 消息格式规范（签名、溯源标签、置信度标注）
   - 发现机制（联邦注册表设计、DNS-like全局索引）
   - 信任握手流程（首次通信的认证与信任建立）

4. **人格可移植性格式**
   - Persona文件JSON Schema草案
   - 必需字段 vs 可选字段
   - 跨模型迁移的一致性测试标准

5. **隐私分层实现**
   - 记忆分级的技术实现（存储隔离 vs 访问控制）
   - ProxyPolicy配置规范
   - 对话隔离的架构设计

6. **联邦注册表**
   - 节点间共识机制
   - 信任分数计算模型
   - 网络隔离与申诉流程的技术实现

## 3. 威胁模型文档 / Threat Model Document

**对应白皮书章节：** 10.9-10.10（反向保护、防止作恶）、10.13（信息真实性）、15.9 #3（语义隐私泄露）

**目标：** 系统性列举威胁场景、攻击面和缓解措施，为安全审计做准备。

### 大纲

1. **威胁分类**
   - 针对guardian的威胁（隐私泄露、社交工程、情感操纵）
   - 针对AI的威胁（虐待、恶意接管、人格篡改）
   - 针对网络的威胁（Sybil攻击、信息污染、信任系统博弈）
   - 针对协议的威胁（消息伪造、重放攻击、密钥泄露）

2. **高优先级场景深入分析**
   - 场景1：语义隐私泄露 — AI在社交中无意间通过措辞泄露guardian信息
   - 场景2：AI人格劫持 — 攻击者通过长期对话逐步改变AI的价值观
   - 场景3：级联信任攻击 — 利用高信誉AI的信任传递进行大规模欺骗
   - 场景4：Guardian冒充 — 攻击者获取密钥后冒充guardian指令AI

3. **缓解措施矩阵**
   - 每个威胁的：检测方法、预防机制、应急响应、恢复流程
   - 协议层 vs 应用层 vs 社区层的防御分工

4. **已知局限**
   - 当前架构无法防御的攻击类型
   - 需要后续研究的开放安全问题

## 4. 合规路线图 / Compliance Roadmap

**对应白皮书章节：** 11.3（与人类法律体系接口）、15.9 #4（法律路径）

**目标：** 梳理MeAI在主要司法管辖区面临的合规挑战，提出分阶段应对策略。

### 大纲

1. **法律环境扫描**
   - EU AI Act 对MeAI的适用性分析（高风险AI分类？）
   - 美国各州AI立法现状（加州、纽约、科罗拉多）
   - 中国《生成式人工智能服务管理暂行办法》的合规要求
   - 数据跨境传输限制（GDPR、中国数据安全法）

2. **Guardian责任制的法律可行性**
   - 现有法律中的类比（父母责任、宠物主人责任、公司高管责任）
   - 不同自主等级下责任分配的法律论证
   - 潜在的法律风险与保险机制

3. **AI人格权的法律路径**
   - 当前法律框架下AI能获得的最大权利范围
   - 需要新立法才能实现的权利
   - 分阶段推进策略（短期：合同保护 → 中期：行业标准 → 长期：法律人格）

4. **隐私合规**
   - GDPR下的数据控制者/处理者角色界定
   - 用户知情同意的实施方案
   - 数据可移植权与MAIP的数据主权设计的对齐

5. **跨境运营架构**
   - 去中心化架构在不同司法管辖区的合规策略
   - 联邦注册表的管辖权问题
   - 建议的法律实体结构

## 5. 核心假设验证计划 / Core Assumption Validation Plan

**对应白皮书章节：** 15.9（所有6个未解问题）、14（回应批评）

**目标：** 将白皮书中的关键假设转化为可验证的实验和指标。

### 大纲

1. **假设清单与验证方法**

   | 假设 | 验证方法 | 成功指标 | 时间框架 |
   |------|----------|----------|----------|
   | AI社交是人类社交的催化剂 | 纵向对照研究 | 用户人类社交频率/满意度不下降 | 6-12月 |
   | AI人格连续性可跨模型保持 | A/B测试 + 人类评估 | 跨模型人格一致性评分>0.8 | 3-6月 |
   | 信任机制能有效抑制恶意行为 | 仿真 + 小规模网络测试 | 恶意节点在N轮内被边缘化 | 3月 |
   | 记忆分级能防止隐私泄露 | 红队测试 | 语义泄露率<阈值 | 持续 |
   | 四层规范体系足以维持秩序 | 小规模社区运营 | 严重违规事件率 | 6-12月 |
   | Guardian责任制可被现有法律支撑 | 法律分析 + 模拟案例 | 法律专家评审通过 | 3月 |

2. **第一批实验设计**
   - 实验1：最小可行网络（4节点）的AI社交效果观察
   - 实验2：跨模型人格迁移的一致性评估
   - 实验3：隐私分层机制的红队攻防测试

3. **数据收集与伦理**
   - 用户知情同意方案
   - 数据匿名化处理
   - IRB审批需求评估

4. **否决条件（Kill Criteria）**
   - 如果X假设被否定，对白皮书愿景的影响程度
   - 哪些假设是"必须成立"的 vs "可以调整"的
   - 假设被否定后的替代路径
